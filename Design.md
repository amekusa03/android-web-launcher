# WebLauncher　設計書

本アプリケーションは、Androidスマートフォンにおいて、特定のWebページをあたかも独立したアプリのように起動・管理するためのアプリケーションである。

## 1. 機能概要

ユーザーは「アプリ一覧（ドロワー）」または「ホーム画面のショートカット」からアイコンをタップすることで、あらかじめ設定されたURLをブラウザで開くことができる。

## 2. アーキテクチャ構成

Androidのシステム制約（動的なActivity生成不可）に基づき、以下の構成を採用する。

1. **静的エントリーポイント（ドロワー用）**

   * `AndroidManifest.xml` にあらかじめ **5つの `activity-alias`** を定義しておく。
   * これらはアプリ一覧（ドロワー）に最大5つまで独立したアイコンとして表示される。
2. **動的ショートカット（ホーム画面用）**
   * `ShortcutManager` を利用し、ホーム画面上に任意の数だけURLショートカットを作成できる。
3. **データ管理**
   * URL、表示名、アイコン設定は `SharedPreferences` または `DataStore` (Jetpack) にて永続化する。

## 3. 実装詳細

### 3.1 アイコンタップ時の挙動

* 設定済みのURLがある場合：**Chrome Custom Tabs** を利用してアプリ内ブラウザ形式でURLを開く。
  * これにより、アプリを離脱せずに高速かつ「アプリらしい」体験を提供する。
* 設定がない（初回起動など）場合：設定画面を表示する。

### 3.2 アイコン長押し（ショートカット/ドロワー共通）

アプリショートカット（Static/Dynamic Shortcuts）の仕組みを利用し、以下のメニューを表示する。

* **編集**: アイコンの種類、表示名、URLを変更する。
* **追加**:
  * ホーム画面に新しいショートカットを作成する（`ShortcutManager`経由）。
* **削除**:
  * ホーム画面のショートカットを削除、またはドロワー用エントリーの設定をリセットする。
  * ※最後のエントリーの場合は削除不可、または無効化。

### 3.3 設定画面（登録・編集）

* **アイコン選択**: 規定の5種類（●印のカラーバリエーション）から選択。
* **表示名**: `label` 設定（1〜15文字程度）。
* **URL**: 開く対象のURL（1〜1024文字）。バリデーションを実施。
* **ブラウザ選択**: 「標準ブラウザで開く」か「Custom Tabsで開く」かを選択可能にする（オプション）。

## 4. 技術スタック

* **Language**: Kotlin
* **UI**: Jetpack Compose (推奨)
* **Browser**: `androidx.browser:browser:1.x.x` (Custom Tabs)
* **Persistence**: `Preferences DataStore`
* **Shortcuts**: `ShortcutManagerCompat`
